Implementation of a single-server client with handler based event dispatch.

There are two ways of creating and connection a client instance:

  1. Blocking mode to be used in it dedicated thread, with asynchronous Handler callbacks, available via the methods without the sync postfix.

  2. Non-Blocking mode, with synchronous Handler callbacks, available via the methods with the sync postfix.

fn connect<A: ToSocketAddrs>(&mut self, handler: &mut Handler<Client>, addr: A) -> Result<(), Error>

Establishes a connection with the server at the specified address by creating a local socket for message sending.

The server must use a compatible connection / packet configuration.

The handler is a struct that implements the Handler trait in order to handle events from the client and its connection.

This method starts the tick loop, blocking the calling thread.
